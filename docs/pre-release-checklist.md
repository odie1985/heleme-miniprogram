# 喝了么 - 提审前检查清单

## 一、账号与基础配置
- [ ] 小程序主体已认证（个人/企业）
- [ ] 小程序名称、头像、简介、服务类目已完善
- [ ] `AppID` 已替换为正式 `wx...`（不是 `touristappid`）
- [ ] 版本号与更新说明已准备（如 `1.0.0`）

## 二、备案与合规
- [ ] 后台“备案”状态已完成或可发布（按公众平台当前要求）
- [ ] 已配置《小程序用户隐私保护指引》
- [ ] 隐私指引声明了实际用到的信息：头像昵称、扫码能力等
- [ ] 若调用隐私相关接口，已接入对应的隐私授权流程
- [ ] 已有用户协议/隐私政策页面（可在“我的”页提供入口）

## 三、功能可用性（当前版本）
- [ ] 首次进入显示“未登录”，点击可登录
- [ ] 登录后昵称/头像可更新，重进后保持登录状态
- [ ] 日历：今天日期始终高亮
- [ ] 日历：有记录日期显示喝酒/健身双色点
- [ ] 快速记录：喝酒/健身两套表单都可提交
- [ ] 好友：点“朋友”胶囊可展开列表
- [ ] 好友：在列表中可“扫码加好友”并成功落库
- [ ] 动态页：能看到自己与好友分享（筛选正常）

## 四、真机与兼容（至少两端）
- [ ] iOS 微信真机完整跑一遍
- [ ] Android 微信真机完整跑一遍
- [ ] 扫码权限弹窗正常（拒绝后有提示，允许后可扫码）
- [ ] 弱网/断网下有友好提示，不崩溃

## 五、提审前代码与资源
- [ ] 删除/隐藏演示逻辑（如“模拟TA分享”仅开发环境保留）
- [ ] 清理无用页面与路由（若不再使用 `friend-add` 可移除）
- [ ] `sitemap.json` 合法
- [ ] 无占位文案、错别字、超大字体/超屏布局

## 六、提审材料准备
- [ ] 功能说明：一句话描述“日历记录喝酒/健身 + 好友分享”

- [ ] 测试账号/说明（如需要）
- [ ] 风险说明：建议补充“理性饮酒”提示

## 七、发布后首日监控
- [ ] 登录成功率
